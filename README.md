# UI Automation Testing Project: Rostelecom 

Данный репозиторий содержит набор автоматизированных тестов для проверки функциональности авторизации и регистрации на портале **Ростелеком ID**. Тесты разработаны с использованием современных подходов к автоматизации (POM) и инструментов тестирования на языке Python.

---

## 🛠 Стек технологий и инструменты

Выбор инструментов обоснован требованиями к стабильности, скорости разработки и чистоте кода:

*   **Python 3** — основной язык программирования. Выбран за лаконичность и мощную экосистему для QA.
*   **PyTest** — фреймворк для запуска тестов. Используются фикстуры (`fixtures`) для управления жизненным циклом браузера и маркеры (`@pytest.mark`) для разделения тестов на группы.
*   **Selenium WebDriver** — инструмент для прямой имитации действий пользователя в браузере.
*   **Page Object Model (POM)** — архитектурный паттерн, разделяющий локаторы и логику страниц. Это упрощает поддержку: при изменении верстки сайта правки вносятся в одном месте.
*   **WebDriverWait (Explicit Waits)** — использование явных ожиданий вместо статических задержек, что делает тесты более стабильными и быстрыми.
*   **Python-dotenv** — безопасное управление тестовыми данными через переменные окружения (файл `.env`).

---

## Техники тест-дизайна

Для обеспечения высокого качества покрытия были применены следующие техники:

1.  **Анализ граничных значений (BVA):** проверка полей «Имя» и «Пароль» на минимально и максимально допустимое количество символов (2, 30, 8, 20).
2.  **Эквивалентное разделение (EP):** разделение входных данных на классы (валидная кириллица vs невалидная латиница; поддерживаемые коды стран +7/+375 vs остальные).
3.  **Предугадывание ошибок (Error Guessing):** сценарии с отключенными Cookies и попытки регистрации на уже существующий Email.
4.  **Тестирование состояний (State Transition):** проверка автоматического переключения табов («Телефон» -> «Почта») при смене формата вводимых данных.

---

## 📋 Проверяемые сценарии

*   **Авторизация:** Успешный вход, автоматическая смена таба, валидация некорректных паролей, очистка полей при переключении вкладок.
*   **Регистрация:** Валидация имени (длина, язык), проверка сложности и совпадения паролей, выбор региона по умолчанию, обработка дубликатов учетных записей.
*   **UI/UX:** Проверка функции «Показать пароль», работа ссылок возврата и информационных сообщений.

## 🚀 Инструкция по запуску

Для запуска тестов на любом компьютере с установленными **Python3** и **PyTest** выполните следующие шаги:

### 1. Подготовка данных
Создайте в корне проекта файл `.env` и заполните его вашими данными (используйте их как секреты):
```env
ok_number=ваш_номер
ok_password_number=ваш_пароль
email=ваш_email
first_name=ваше_имя
last_name=ваще_фамилия
# ... и другие переменные из тестов
``` 

### 2. Установка зависимостей

Откройте терминал в папке проекта и выполните команду для установки необходимых библиотек: 
```bash
pip install pytest selenium python-dotenv 
```
### 3. Запуск тестов

Все тесты:


```bash
pytest
```

Только тесты регистрации (позитивные):
```bash
pytest -m positive_reg
pytest -m positive_auth
```

Только негативные тесты авторизации:
```bash
pytest -m negative_auth
pytest -m negative_reg
```
